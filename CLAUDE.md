# Project Rules

## 源码优先原则

- **src 目录**是源码权威来源
- .claude 目录下的代码是 src 目录的镜像/输出位置
- 修改代码时，以 src 目录为准
- 修改完成后，自动同步到 .claude 目录

## 同步规则

当修改 src 目录下的文件后，相应文件应同步到：
- `.claude/skills/capture-you/` — capture-you 技能文件

同步操作由 Claude Code 自动执行。

## TDD 开发原则

- **TDD 优先**：新增功能或修改代码时，先写测试用例，再实现功能
- **测试覆盖率**：确保新功能有对应的单元测试，修改已有逻辑时同步更新相关测试
- **回归保护**：所有测试用例应通过，确保添加新功能和改动不会影响已有业务逻辑
- **测试驱动重构**：重构前先确保有测试覆盖，重构后测试仍应通过
