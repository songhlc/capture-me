# Project Rules

## 源码优先原则（强制）

**任何代码修改必须遵循以下流程，绝不允许直接修改 .claude 目录：**

1. **修改源码**：所有代码修改必须在 `src/` 目录下进行
2. **同步发布**：修改完成后，手动将改动同步到目标位置
3. **禁止直接修改**：严禁直接修改 `.claude` 目录下的任何文件

违反此规则视为严重错误。

## 源码目录结构

| 源码位置 | 目标位置 | 说明 |
|----------|----------|------|
| `src/skills/capture-me/` | `~/.claude/skills/capture-me/` | capture-me 技能源码 |

## 同步操作

修改 `src/` 目录下的文件后，必须执行同步命令：

```bash
# 同步 capture-me 相关文件
cp src/skills/capture-me/*.js ~/.claude/skills/capture-me/
```

同步是**强制的**，未同步的改动不会被执行。

## TDD 开发原则

- **TDD 优先**：新增功能或修改代码时，先写测试用例，再实现功能
- **测试覆盖率**：确保新功能有对应的单元测试，修改已有逻辑时同步更新相关测试
- **回归保护**：所有测试用例应通过，确保添加新功能和改动不会影响已有业务逻辑
- **测试驱动重构**：重构前先确保有测试覆盖，重构后测试仍应通过
